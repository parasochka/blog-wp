# Миграция: FSE block-тема → классическая PHP-тема «NOW»

Как переехать со старой реализации на новую и что делать дальше, чтобы всё
работало. Коротко: **новая тема — классическая PHP на дизайн-системе `_ds/`**;
контент, категории, меню и плагины остаются, деплой — через WP Pusher из `main`.

---

## 0. TL;DR

1. Влить ветку `claude/cloud-design-wordpress-integration-tznp9j` в `main`.
2. WP Pusher → тема `blog-wp` → **Update** (подтянет `main`).
3. Appearance → Themes → активировать **«NOW — NowPlix Blog» v1.0.0**.
4. Settings → Permalinks → `/%category%/%postname%/` → Save.
5. (Опц.) отключить *Easy Table of Contents* и *Inline Related Posts* — у темы
   есть свои TOC и «More stories».

Готово: главная, категории, статьи, поиск, 404 рендерятся 1-в-1 с дизайном на
твоём реальном контенте.

---

## 1. Что изменилось и почему

| | Было (старое) | Стало (новое) |
| --- | --- | --- |
| Тип темы | FSE block theme | Классическая PHP-тема |
| Источник стилей | `theme.json`-пресеты + блочная разметка | `_ds/…/tokens/*.css` подключены как есть — **единственный источник правды** |
| Шаблоны | `templates/*.html` (блоки) | `*.php`, дословно повторяют экраны хэндофа |
| `theme.json` | большой токен-мост | **удалён** (классике не нужен) |

Почему меняли: два расходящихся источника токенов (tokens.css vs theme.json)
дрейфовали; блочная разметка теряла свечения/градиенты/hover; пиксель-в-пиксель
не сходился. Теперь один источник токенов, редактируется кодом и AI.

---

## 2. Деплой (WP Pusher)

WP Pusher уже установлен и настроен на `parasochka/blog-wp`, ветка **main**,
install from subdirectory — **`theme`** (тема живёт в `theme/`; docs и tools
из корня репо на сервер не попадают).

1. **Влить ветку в `main`** (PR или прямой merge). WP Pusher деплоит именно
   `main` — пока правки в фиче-ветке, на сайте старая тема.
2. WP Pusher → **Themes → blog-wp → Update** (или автоматически, если включён
   Push-to-Deploy).
3. **Appearance → Themes → активировать** «NOW — NowPlix Blog».

**Откат:** переключиться на прежнюю тему в Appearance → Themes, либо откатить
`main` на предыдущий коммит и нажать Update в WP Pusher.

## 3. Настройки WordPress после активации

- **Reading:** Settings → Reading → *Your latest posts* (главная = `home.php`:
  hero + featured + рельсы). Сейчас уже так. ✓
- **Permalinks:** Settings → Permalinks → Custom `/%category%/%postname%/` → Save.
  (Через MCP настройки постоянных ссылок не пишутся — выставить вручную.)
- **Site Logo:** необязателен — тема сама подставляет
  `assets/img/logo-now-glass.png`. Хочешь из админки — Customize → Logo; тема
  отдаст приоритет загруженному (поддержка `custom-logo` уже есть).
- **Меню:** тема регистрирует локацию **Primary (header)**. Без назначенного
  меню в шапке — *Home + первые 4 категории*. Чтобы курировать: Appearance →
  Menus → назначить «Main Menu» на локацию **Primary** (доступно только после
  активации темы).
- **Featured на главной:** featured-блок = самый свежий пост. Чтобы закрепить
  конкретный — сделать пост **Sticky** (Publish → Stick to the front page).

## 4. Плагины: что оставить, что выключить

Активны: AI · AI Provider for OpenAI · Easy MCP AI · Easy Table of Contents ·
Inline Related Posts · Instant Images · Rank Math SEO · WP Pusher.

- **Оставить:** WP Pusher (деплой), Easy MCP AI (коннектор AI/MCP), Rank Math SEO,
  Instant Images (удобно тянуть featured из Unsplash/Pexels для новых постов),
  AI / AI Provider — по желанию.
- **Отключить (рекомендуется):**
  - **Easy Table of Contents** — у темы свой TOC «On this page» в сайдбаре
    статьи (авто-оглавление + scrollspy, `now.js`). Иначе два оглавления.
  - **Inline Related Posts** — у темы свой блок «More stories» (related по
    категории) внизу статьи. Иначе дубли.

  (Либо оставить плагины, но отключить их вывод в записях в настройках плагина.)

## 6. Дальнейший pipeline

```
theme/ (_ds/now + *.php)  ──push→ main──►  WP Pusher  ──►  WordPress (тема)
       (код, git)                     (Update)         рендерит контент
контент (посты/категории/меню/медиа) ──────────────►  WordPress (админка / MCP)
```

- **Дизайн/токены/шаблоны** → правки в репо → push в `main` → WP Pusher тянет.
  Правка `--primary-500` в `theme/_ds/now/tokens/colors.css` перекрашивает весь сайт.
- **Контент** → в WordPress (админка или через MCP), **не** в git.
- **Ре-экспорт дизайн-системы** Claude Design → положить содержимое нового
  экспорта поверх `theme/_ds/now/` (тот же путь) → чистый diff, без ручной
  перетрансляции.

Подробные правила разработки темы — в [`CLAUDE.md`](./CLAUDE.md).
